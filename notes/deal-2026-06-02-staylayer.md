## Red flags

### 🔴 Critiques

1. **Revenus non existants / non vérifiables** — Pre-launch total. Aucune donnée financière. Le listing ne peut montrer aucune preuve de traction commerciale par construction.

2. **Delta vendeur/Ahrefs >100%** — Les métriques SEO affichées dans le listing (Authority Score 23, 2.83K referring domains) concernent le domaine partagé Vercel, pas StayLayer. Le produit n'a aucun actif SEO propre.

3. **Aucun domaine custom** — Les top keywords "Vercel.App / Inurl:Vercel.App" confirment que le produit est déployé sur *.vercel.app. Il n'y a pas de domaine réel à acquérir dans la vente (non mentionné dans les assets inclus).

### 🟡 Attention

4. **Business < 12 mois** — 11 mois d'existence, aucune validation marché.

5. **Texte marketing génératif suspect** — Description très polie et générique ("The strongest value of StayLayer is not that it exists in a market without competition..."), cohérente avec du texte IA, aucune preuve concrète de la qualité du code.

6. **Multiple implicitement inacceptable** — Sans revenu, le seul référentiel est la valeur du code. $4K pour un SaaS hospitalité non lancé est arbitraire et non justifié par des métriques objectives.

7. **Raison de vente floue** — "I brought it to the point where the product architecture is clear" = le vendeur abandonne avant la partie difficile (commercialisation) sans démontrer que le produit fonctionne.

### ℹ️ Info

- Stack non précisé (présumé Next.js/React vu Vercel)
- Marché hospitalité compétitif (Lodgify, Hostfully, etc.)
- Churn 0% = 0 client, pas une métrique positive
